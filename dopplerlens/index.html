How DopplerLens works

DopplerLens uses the same family of algorithms as dnstwist — character omission, duplication, transposition, keyboard-adjacent substitution, homoglyph substitution, character insertion, hyphenation, TLD swaps, common prefixes/suffixes, and bitsquatting. Each algorithm produces variations of your domain that humans might mistype or fail to notice.

For each candidate, DopplerLens runs DNS lookups via Google's public DNS-over-HTTPS resolver to see if the domain is registered. Live candidates get deeper analysis: nameservers, mail servers, web server reachability, and a favicon hash comparison against your domain.

Confidence and readiness

Each finding shows a confidence (how sure we are it's malicious) and a readiness tag describing how operationally prepared the domain is to host an attack: Ready means web + mail are both active, Staged means web only, Registered means DNS only. A "Ready" domain can phish your users today; a "Registered" one needs the attacker to do more work first.

Baseline tracking and snoozing

Each scan is compared against your previous scan of the same domain. Domains seen for the first time are flagged as new. You can snooze any finding you've already triaged (defensive registrations you own, expired threats, known false positives) — snoozed domains stay visible but don't clutter the New tab on subsequent scans. Baselines and snoozes are stored only in your browser (localStorage) and are never sent anywhere.

Safe investigation

Visiting a suspected phishing site from a corporate browser is risky — the page may host browser exploits or session-stealing scripts. DopplerLens prefers urlscan.io (sandboxed rendering with screenshot), web.archive.org (cached static view), and VirusTotal over direct visits. Direct "Open" is available but warns first.

Abuse reports

For high-confidence findings, DopplerLens generates a structured abuse report you can paste into the appropriate channels: registrar abuse contact, hosting provider abuse contact, Google Safe Browsing, Microsoft SmartScreen, APWG, and Cisco Talos. Before reporting, please verify the site is actually impersonating you — false-positive abuse reports damage your credibility with takedown teams.

Honest limits

DopplerLens does not detect IDN homograph attacks using non-Latin Unicode reliably from the browser. Favicon comparison only works when the suspect domain serves its favicon with permissive CORS headers, which most don't, so absence of a favicon match is not exonerating evidence. The DNS-based liveness check can produce false positives for shared parking infrastructure. Keyboard-adjacency uses a US QWERTY layout.
