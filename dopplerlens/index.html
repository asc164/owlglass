How DopplerLens works

DopplerLens uses the same family of algorithms as dnstwist — character omission, duplication, transposition, keyboard-adjacent substitution, homoglyph substitution, character insertion, hyphenation, TLD swaps, common prefixes/suffixes, and bitsquatting.

For each candidate, DopplerLens runs DNS lookups via Google's public DNS-over-HTTPS resolver. Live candidates get deeper analysis: nameservers, mail servers, web server reachability, and a favicon hash comparison.

Confidence and readiness

Each finding shows a confidence and a readiness tag: Ready means web + mail are both active, Staged means web only, Registered means DNS only.

Screenshots via urlscan.io

For Likely-phishing and Suspicious findings, DopplerLens queries urlscan.io's public search API for an existing screenshot of the domain. We don't submit new scans — only show what's already in their public archive. If no prior scan exists, you'll see a "Submit to urlscan" link instead. Click any thumbnail to enlarge.

Scan history and baselines

DopplerLens keeps your last 10 scans in browser storage. Click any chip in the history strip to instantly restore that scan without re-fetching. Each scan also stores a baseline for diffing the next scan.

Honest limits

DopplerLens does not detect IDN homograph attacks using non-Latin Unicode reliably from the browser. Favicon comparison only works when the suspect domain serves its favicon with permissive CORS headers. Keyboard-adjacency uses a US QWERTY layout. urlscan thumbnails only appear for domains with prior public scans.
