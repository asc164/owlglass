How DopplerLens works

DopplerLens uses the same family of algorithms as dnstwist — character omission, duplication, transposition, keyboard-adjacent substitution, homoglyph substitution, character insertion, hyphenation, TLD swaps, common prefixes/suffixes, and bitsquatting. Each algorithm produces variations of your domain that humans might mistype or fail to notice.

For each candidate, DopplerLens runs DNS lookups via Google's public DNS-over-HTTPS resolver to see if the domain is registered. Live candidates get deeper analysis: nameservers, mail servers, web server reachability, and a favicon hash comparison against your domain.

Baseline tracking

Each scan is compared against your previous scan of the same domain. Domains seen for the first time are flagged as new. Newly registered phishing infrastructure is the most actionable signal this tool produces. Baselines are stored only in your browser (localStorage) and are never sent anywhere.

Abuse reports

For high-confidence findings, DopplerLens generates a structured abuse report you can paste into the appropriate channels: registrar abuse contact, hosting provider abuse contact, Google Safe Browsing, Microsoft SmartScreen, APWG, and Cisco Talos. The report includes domain ownership signals, infrastructure evidence, and recommended actions — formatted for human reviewers at takedown teams.

Honest limits

DopplerLens does not detect IDN homograph attacks using non-Latin Unicode reliably from the browser. Favicon comparison only works when the suspect domain serves its favicon with permissive CORS headers, which most don't, so absence of a favicon match is not exonerating evidence. The DNS-based liveness check can produce false positives for shared parking infrastructure.
